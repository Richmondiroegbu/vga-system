## Context — THE MOST CRITICAL IMPLEMENTATION IN VGA

The Temporal Engine is the architectural heart of VGA's video quality. It implements **autoregressive segment generation** — each video segment is generated by conditioning on the previous 5 frames (TemporalBuffer), creating temporal continuity that prevents motion resets between segments.

### TEMPORAL ENFORCEMENT BLOCK v17.2 (READ FIRST — MANDATORY)

```
MANDATORY:
  - Sequential execution: one segment at a time, in strict order
  - Buffer-driven conditioning: every segment conditioned on B_t (exactly 5 frames)
  - Autoregressive chaining: Segment_{n+1} depends ONLY on B_n = last_5_frames(Segment_n)
  - TemporalBuffer ordered as frames[0]=t-4 ... frames[4]=t (most recent)
  - SVI MUST operate ONLY inside the autoregressive loop body

FORBIDDEN (any of these = HARD FAILURE → CriticalPipelineError):
  - Batch segment generation (generating all segments in one call)
  - Segment merging after independent generation
  - Single-frame conditioning for Segment_2 or later
  - Precomputed segment lists passed to TemporalEngine
  - External injection of segments into the autoregressive loop
  - Static LoRA weight during SVI generation
```

### Generation Flow
```
Segment_1: Wan2.2 (video_segment_generator.py) ← init_image from S-07
           ↓
           TemporalBufferManager.init(segment_1) → B_1 (5 frames)
           ↓
For n=2..N: TemporalEngine (temporal_engine.py)
           ↓
           B_{n-1} → TemporalBufferManager.encode() → latents (5-frame tensor)
           ↓
           SVIScheduler → dynamic LoRA weight per noise phase
           ↓
           SVI.generate(latents=latents, lora_high=..., lora_low=..., cfg=[5.0-6.0])
           ↓
           Segment_n → CLIPValidator (≥0.93) + ContinuityValidator (≥0.85) + TemporalConsistencyValidator
           ↓
           TemporalBufferManager.update(B_{n-1}, Segment_n) → B_n
           ↓
           MotionStateTracker.estimate(B_n) → MotionState
           ↓
           context.evolve() → new context with motion_state + temporal_state updated
```

```python
"""
TemporalBufferManager — SOLE owner of TemporalBuffer lifecycle.
RULE-86: Buffer MUST contain exactly 5 frames at all times.
FR-912: BUFFER_SIZE=5 STRICT — no deviation.
Spec: VGA SRD v17.2 §18.4 (TemporalBuffer Lifecycle Contract — authoritative)
"""
from __future__ import annotations
import torch
from torch import Tensor
from dataclasses import dataclass
from typing import List, Optional
from vga.config.settings import settings
from vga.core.exceptions import TemporalBufferError
from vga.core.logger import get_logger

log = get_logger(__name__)

BUFFER_SIZE = settings.TEMPORAL_BUFFER_SIZE  # = 5, immutable

@dataclass(frozen=True)
class TemporalBuffer:
    """
    Fixed-size rolling window of exactly 5 frames.
    
    Properties (FROM SPEC — all mandatory):
    - size = 5 (strict; no deviation permitted)
    - ordered: frames[0]=t-4, frames[1]=t-3, ..., frames[4]=t (most recent)
    - all frames: same resolution H×W
    - all frames: same color space (consistent RGB throughout)
    - all frames: normalized to [0.0, 1.0] range
    - ALWAYS CPU-resident between segment generation steps
    - Only moves to GPU during encode() — immediately returned to CPU
    
    Lifecycle States: INIT → UPDATE → VALIDATION → ENCODE
    """
    frames: Tensor        # shape: (5, C, H, W), dtype: float32, CPU
    timestamps: tuple[float, ...]   # exactly 5 timestamps, strictly increasing
    scene_id: str
    segment_index: int    # which segment this buffer was built from
    motion_vector: Optional[Tensor] = None  # from MotionStateTracker
    
    def __post_init__(self):
        """Validate buffer invariants on creation."""
        TemporalBufferManager._assert_buffer_size(self)


class TemporalBufferManager:
    """
    Manages TemporalBuffer lifecycle: init, update, encode.
    
    AUTHORITY: This is the SOLE component that creates or modifies TemporalBuffers.
    No other component may construct a TemporalBuffer directly.
    
    Device Rule (FROM SPEC — mandatory):
      - Buffer frames are ALWAYS stored as CPU tensors
      - encode() is the ONLY method that transfers frames to GPU
      - After encode(): tensor immediately transferred back to CPU; GPU memory released
    """
    
    # Normalization constants (FIXED — must not be computed per-call)
    _NORM_MEAN = torch.tensor([0.485, 0.456, 0.406]).view(3, 1, 1)
    _NORM_STD = torch.tensor([0.229, 0.224, 0.225]).view(3, 1, 1)
    
    @classmethod
    def init(cls, segment_1: "VideoSegment", scene_id: str) -> TemporalBuffer:
        """
        Initialize buffer from Segment_1 (called once by VideoSegmentGenerator).
        
        Steps:
          1. Extract last 5 frames from segment_1
          2. Assert frame count == 5 (raises TemporalBufferError if not)
          3. Normalize frames
          4. Assert temporal order of timestamps
          5. Create TemporalBuffer on CPU
          6. Assert buffer size == 5
          7. Log temporal_buffer_initialized
          8. Return buffer
        
        Raises: TemporalBufferError if segment_1 has < 5 frames
        """
        frames = cls._extract_last_5_frames(segment_1.frames_tensor)
        timestamps = cls._extract_timestamps(segment_1, count=5)
        cls._assert_temporal_order(list(timestamps))
        normalized = cls._normalize_frames(frames)
        
        buffer = TemporalBuffer(
            frames=normalized.cpu(),
            timestamps=tuple(timestamps),
            scene_id=scene_id,
            segment_index=1,
        )
        cls._assert_buffer_size(buffer)
        
        log.info("temporal_buffer_initialized", scene_id=scene_id, frame_count=5)
        return buffer
    
    @classmethod
    def update(
        cls,
        current_buffer: TemporalBuffer,
        new_segment: "VideoSegment",
    ) -> TemporalBuffer:
        """
        Rolling update: replace buffer with last 5 frames of new_segment.
        
        Steps:
          1. Extract last 5 frames from new_segment
          2. Normalize
          3. Assert timestamps strictly increasing
          4. Create new TemporalBuffer (old buffer discarded)
          5. Assert new buffer size == 5
          6. Log temporal_buffer_updated
          7. Return new buffer (old buffer can be GC'd)
        
        Raises: TemporalBufferError if constraints violated
        """
        frames = cls._extract_last_5_frames(new_segment.frames_tensor)
        timestamps = cls._extract_timestamps(new_segment, count=5)
        cls._assert_temporal_order(list(timestamps))
        normalized = cls._normalize_frames(frames)
        
        new_buffer = TemporalBuffer(
            frames=normalized.cpu(),
            timestamps=tuple(timestamps),
            scene_id=current_buffer.scene_id,
            segment_index=current_buffer.segment_index + 1,
        )
        cls._assert_buffer_size(new_buffer)
        
        log.info(
            "temporal_buffer_updated",
            scene_id=current_buffer.scene_id,
            segment_index=new_buffer.segment_index,
            frame_count=5,
        )
        return new_buffer
    
    @classmethod
    def encode(cls, buffer: TemporalBuffer, device: str = "cuda") -> Tensor:
        """
        Encode buffer frames as latents for SVI conditioning.
        
        CRITICAL DEVICE RULE:
          - frames.to(device) ONLY inside this method
          - latents transferred back to CPU before return
          - GPU tensors MUST NOT persist beyond this method call
        
        Returns: CPU tensor of shape (5, C', H', W') — multi-frame latents
        Raises: TemporalBufferError if buffer has wrong shape
        """
        cls._assert_buffer_size(buffer)
        assert buffer.frames.is_cpu, "Frames must be CPU-resident before encode()"
        
        # Transfer to GPU for encoding
        frames_gpu = buffer.frames.to(device)
        
        # Encode as latents (VAE encode or direct frame conditioning)
        # Use SVI's built-in frame encoder — this is the SVI conditioning input
        latents = cls._encode_frames_to_latents(frames_gpu)
        assert latents.shape[0] == BUFFER_SIZE, f"Latent count {latents.shape[0]} ≠ {BUFFER_SIZE}"
        
        # Return to CPU immediately
        latents_cpu = cls._to_cpu(latents)
        del frames_gpu, latents
        
        log.info("temporal_buffer_encoded", latent_shape=list(latents_cpu.shape))
        return latents_cpu
    
    @classmethod
    def _extract_last_5_frames(cls, video_tensor: Tensor) -> Tensor:
        """Extract last 5 frames from segment. Raises TemporalBufferError if < 5 frames."""
        total_frames = video_tensor.shape[0]
        if total_frames < BUFFER_SIZE:
            raise TemporalBufferError(
                f"Segment has {total_frames} frames — need at least {BUFFER_SIZE} (RULE-86)",
                frame_count=total_frames,
                required=BUFFER_SIZE,
            )
        return video_tensor[-BUFFER_SIZE:].clone()  # (5, C, H, W)
    
    @classmethod
    def _normalize_frames(cls, frames: Tensor) -> Tensor:
        """
        Normalize frames to [0.0, 1.0] using FIXED constants (MUST be consistent).
        NEVER compute normalization parameters per-call.
        """
        if frames.max() > 1.0:
            frames = frames / 255.0  # uint8 → float32 [0,1]
        # Apply ImageNet normalization
        return (frames - cls._NORM_MEAN) / cls._NORM_STD
    
    @classmethod
    def _assert_temporal_order(cls, timestamps: list[float]) -> None:
        """Validate timestamps are strictly increasing. Raises TemporalBufferError."""
        for i in range(1, len(timestamps)):
            if timestamps[i] <= timestamps[i - 1]:
                raise TemporalBufferError(
                    f"Temporal order violation: timestamps[{i}]={timestamps[i]} "
                    f"<= timestamps[{i-1}]={timestamps[i-1]}"
                )
    
    @staticmethod
    def _assert_buffer_size(buffer: TemporalBuffer) -> None:
        """
        HARD RULE: Buffer must have exactly 5 frames.
        Called from EVERY method that creates or returns a TemporalBuffer.
        Raises: TemporalBufferError immediately on violation.
        """
        frame_count = buffer.frames.shape[0]
        if frame_count != BUFFER_SIZE:
            raise TemporalBufferError(
                f"TemporalBuffer MUST have exactly {BUFFER_SIZE} frames (RULE-86). "
                f"Got {frame_count}. This is a CRITICAL violation — pipeline halts.",
                frame_count=frame_count,
                required=BUFFER_SIZE,
            )
    
    @staticmethod
    def _to_cpu(tensor: Tensor) -> Tensor:
        return tensor.detach().cpu()
    
    @staticmethod
    def _encode_frames_to_latents(frames_gpu: Tensor) -> Tensor:
        """
        Encode 5 frames as latents using SVI's conditioning mechanism.
        frames_gpu: (5, C, H, W) on GPU
        Returns: (5, C', H', W') latents on GPU (caller transfers to CPU)
        """

    def _extract_timestamps(segment: "VideoSegment", count: int) -> list[float]:
```

```python
"""
SVIScheduler — Noise-aware LoRA weight scheduler for SVI Pro 2.
RULE-86: Dynamic LoRA weights per diffusion phase; STATIC WEIGHTS FORBIDDEN.
FR-931–FR-935: High/mid/low noise phase scheduling.
FR-936: SVI CFG MUST be clamped to [5.0, 6.0].
Spec: VGA SRD v17.2 §3.68
"""
from vga.config.settings import settings
from vga.core.exceptions import SVICFGViolationError, SVISchedulerViolationError
from vga.models.enums import TemporalPhase

class SVIScheduler:
    """
    Implements dynamic LoRA weight scheduling per diffusion timestep phase.
    
    Phase mapping:
      HIGH_NOISE:  t > threshold_high (0.67*T) → weight = 0.6 (structure + motion)
      MID_NOISE:   threshold_mid < t <= threshold_high → weight = 0.5
      LOW_NOISE:   t <= threshold_mid (0.33*T) → weight = 0.4 (detail preservation)
    
    CRITICAL: The scheduler MUST be called per diffusion timestep.
    Static LoRA weight assignment (e.g., lora_weight=0.5 for all steps) is FORBIDDEN.
    """
    
    def __init__(
        self,
        total_steps: int,
        cfg: float = settings.SVI_CFG_DEFAULT,
        is_critical_segment: bool = False,
    ):
        self._validate_cfg(cfg)
        self.total_steps = total_steps
        self.cfg = cfg
        self.is_critical = is_critical_segment
        
        # Thresholds (fraction of total steps)
        self._high_noise_step = int(settings.HIGH_NOISE_FRACTION * total_steps)
        self._mid_noise_step = int(settings.MID_NOISE_FRACTION * total_steps)
    
    def get_lora_weight(self, timestep_index: int) -> float:
        """
        Return dynamic LoRA weight for the given diffusion timestep.
        MUST be called per timestep — never precomputed as a static value.
        
        Args:
            timestep_index: current diffusion step (0=start, T=end)
        Returns:
            LoRA weight: 0.6 (high noise), 0.5 (mid), 0.4 (low)
        """
        phase = self.get_phase(timestep_index)
        return {
            TemporalPhase.HIGH_NOISE: settings.LORA_WEIGHT_HIGH_NOISE,  # 0.6
            TemporalPhase.MID_NOISE: settings.LORA_WEIGHT_MID_NOISE,    # 0.5
            TemporalPhase.LOW_NOISE: settings.LORA_WEIGHT_LOW_NOISE,    # 0.4
        }[phase]
    
    def get_phase(self, timestep_index: int) -> TemporalPhase:
        """Determine noise phase for the given timestep."""
        # Note: diffusion works t=T→0 (high noise first, low noise last)
        # Reverse mapping: early steps = high noise
        remaining = self.total_steps - timestep_index
        if remaining > self._high_noise_step:
            return TemporalPhase.HIGH_NOISE
        elif remaining > self._mid_noise_step:
            return TemporalPhase.MID_NOISE
        else:
            return TemporalPhase.LOW_NOISE
    
    def get_steps(self) -> int:
        """Return step count based on segment criticality. FR-937."""
        if self.is_critical:
            return settings.STEPS_CRITICAL  # 30–50
        return settings.STEPS_STANDARD       # 4–8 spec says, use 30 as quality minimum
    
    @staticmethod
    def _validate_cfg(cfg: float) -> None:
        """Assert CFG in [5.0, 6.0]. FR-936."""
        if not (settings.SVI_CFG_MIN <= cfg <= settings.SVI_CFG_MAX):
            raise SVICFGViolationError(cfg_value=cfg)
    
    def build_generation_params(self, timestep_index: int) -> dict:
        """
        Build complete SVI generation parameters for a single diffusion step.
        Returns dict with: lora_weight, cfg, phase.
        Used to pass parameters to SVI inference script.
        """
        lora_weight = self.get_lora_weight(timestep_index)
        return {
            "lora_weight_high_noise": lora_weight if self.get_phase(timestep_index) == TemporalPhase.HIGH_NOISE else None,
            "lora_weight_low_noise": lora_weight if self.get_phase(timestep_index) == TemporalPhase.LOW_NOISE else None,
            "cfg": self.cfg,
            "phase": self.get_phase(timestep_index).value,
        }
```

## vga/temporal/temporal_engine.py

```python
"""
TemporalEngine — THE ONLY component that executes the SVI autoregressive loop.
TEMPORAL ENGINE AUTHORITY v17.2: All other components are FORBIDDEN from:
  - Controlling segment iteration
  - Updating TemporalBuffer
  - Invoking SVI
  - Managing autoregressive generation flow

RULE-87: Segment_{n+1} MUST be conditioned on Segment_{n} via TemporalBuffer.
RULE-107: SVI generation MUST be explicit per-segment for loop. Batch FORBIDDEN.
Spec: VGA SRD v17.2 §5.28, §18.1, §18.3
"""
from __future__ import annotations
from typing import List
from vga.state.immutable_context import ImmutableContext
from vga.temporal.temporal_buffer_manager import TemporalBufferManager, TemporalBuffer
from vga.temporal.svi_scheduler import SVIScheduler
from vga.temporal.motion_state_tracker import MotionStateTracker, MotionState
from vga.temporal.temporal_retry_controller import TemporalRetryController
from vga.validation.clip_validator import CLIPValidator
from vga.config.settings import settings
from vga.core.exceptions import (
    TemporalSegmentFailureError, TemporalBufferError, AutoregressiveViolationError
)
from vga.core.logger import get_logger

log = get_logger(__name__)

class TemporalEngine:
    """
    Executes autoregressive segment generation for Segments 2..N.
    
    Segment_1 is generated by VideoSegmentGenerator (Wan2.2).
    THIS class handles Segments 2..N using SVI Pro 2.
    
    THE AUTOREGRESSIVE LOOP:
    for n in range(2, N+1):
        1. Assert buffer has exactly 5 frames (RULE-86)
        2. Encode buffer → multi-frame latents (NOT single frame)
        3. Get SVIScheduler (dynamic LoRA per noise phase)
        4. Invoke SVI via subprocess (isolated svi_wan22 conda env)
        5. Validate: CLIP ≥ 0.93, continuity ≥ 0.85, buffer size integrity
        6. Update TemporalBuffer with new segment
        7. Estimate MotionState from updated buffer
        8. Evolve context with motion_state + temporal_state
        9. Log TemporalBufferRecord + MotionStateRecord + SVIGenerationRecord
    """
    
    def __init__(
        self,
        buffer_manager: TemporalBufferManager,
        motion_tracker: MotionStateTracker,
        retry_controller: TemporalRetryController,
        clip_validator: CLIPValidator,
    ):
        self.buffer_manager = buffer_manager
        self.motion_tracker = motion_tracker
        self.retry_controller = retry_controller
        self.clip_validator = clip_validator
    
    def generate_segments(
        self,
        segment_1: "VideoSegment",
        initial_buffer: TemporalBuffer,
        total_segments: int,
        segment_prompts: List[str],
        composition_plan: "CompositionPlanSchema",
        context: ImmutableContext,
    ) -> tuple[List["VideoSegment"], ImmutableContext]:
        """
        THE AUTOREGRESSIVE LOOP — Generates Segments 2..N.
        
        VALIDATION PROPAGATION CONTRACT (v17.1):
        Every segment MUST pass ALL THREE validators before being committed:
          1. ContinuityValidator — continuity_score ≥ SEGMENT_CONTINUITY_MIN
          2. CLIPValidator       — clip_score ≥ CLIP_IDENTITY_THRESHOLD (0.93)
          3. TemporalConsistencyValidator — buffer size = 5; no resolution mismatch
        
        Failure of ANY validator → TemporalRetryController.retry()
        If retries exhausted → raise TemporalSegmentFailureError
        """
        segments = [segment_1]  # Segment_1 already generated by VideoSegmentGenerator
        current_buffer = initial_buffer
        current_context = context
        
        # AUTOREGRESSIVE LOOP — RULE-107: explicit for loop, never batch
        for n in range(2, total_segments + 1):
            # Assert buffer invariants BEFORE generating next segment
            self.buffer_manager._assert_buffer_size(current_buffer)
            
            segment_n = self._generate_single_segment_with_retry(
                n=n,
                buffer=current_buffer,
                prompt=segment_prompts[n - 1],
                composition_plan=composition_plan,
                context=current_context,
                is_critical=(n <= 2),  # first segments are critical
            )
            
            # All 3 validators must pass before committing
            self._validate_segment_triple(
                segment=segment_n,
                buffer=current_buffer,
                context=current_context,
                segment_index=n,
            )
            
            segments.append(segment_n)
            
            # Update buffer (rolling: last 5 frames of Segment_n become B_n)
            current_buffer = self.buffer_manager.update(current_buffer, segment_n)
            
            # Estimate motion state from updated buffer
            motion_state = self.motion_tracker.estimate(current_buffer)
            
            # Evolve context with BOTH motion_state and temporal_state
            from vga.models.schemas import TemporalState
            current_context = current_context.evolve(
                motion_state=motion_state,
                temporal_state=TemporalState(
                    buffer=current_buffer,
                    segment_index=n,
                ),
            )
            
            # Log records
            self._log_segment_records(n, segment_n, current_buffer, motion_state, context)
        
        return segments, current_context
    
    def _generate_single_segment_with_retry(
        self,
        n: int,
        buffer: TemporalBuffer,
        prompt: str,
        composition_plan: "CompositionPlanSchema",
        context: ImmutableContext,
        is_critical: bool,
    ) -> "VideoSegment":
        """
        Generate a single segment (n) using SVI Pro 2 with retry logic.
        
        CRITICAL:
        - buffer MUST have exactly 5 frames (asserted before call)
        - SVI called via subprocess (svi_wan22 conda env)
        - Never called in batch — always single segment
        - CFG clamped to [5.0, 6.0]
        """
        scheduler = SVIScheduler(
            total_steps=scheduler.get_steps(),
            cfg=settings.SVI_CFG_DEFAULT,
            is_critical_segment=is_critical,
        )
        
        for attempt in range(settings.TEMPORAL_MAX_RETRIES_PER_SEGMENT):
            try:
                # Encode buffer → multi-frame latents (NOT single frame — RULE-87)
                latents = self.buffer_manager.encode(buffer)
                assert latents.shape[0] == settings.TEMPORAL_BUFFER_SIZE, \
                    f"Latent count {latents.shape[0]} ≠ 5 (RULE-87: single-frame conditioning FORBIDDEN)"
                
                # Invoke SVI via subprocess (isolated environment)
                segment = self._invoke_svi_subprocess(
                    init_latents=latents,
                    prompt=prompt,
                    scheduler=scheduler,
                    composition_plan=composition_plan,
                    segment_index=n,
                )
                return segment
                
            except Exception as e:
                if attempt == settings.TEMPORAL_MAX_RETRIES_PER_SEGMENT - 1:
                    raise TemporalSegmentFailureError(
                        scene_id=context.scene_id,
                        segment_id=n,
                    ) from e
                # Adjust parameters on retry
                scheduler = self.retry_controller.adjust(scheduler, e, attempt)
        
        raise TemporalSegmentFailureError(context.scene_id, n)
    
    def _invoke_svi_subprocess(
        self,
        init_latents: "Tensor",
        prompt: str,
        scheduler: SVIScheduler,
        composition_plan: "CompositionPlanSchema",
        segment_index: int,
    ) -> "VideoSegment":
        """
        Invoke SVI inference via subprocess in the svi_wan22 conda env.
        
        The svi_wan22 branch requires PyTorch 2.7.1 + cu128 (DiffSynth 2.0).
        This is INCOMPATIBLE with the main VGA cu124 stack.
        ALWAYS call via subprocess with SVI_WAN22_PYTHON interpreter.
        
        Command:
          {SVI_WAN22_PYTHON} {SVI_REPO_PATH}/inference_svi_2.0.py \\
            --lora_path_high {SVI_HIGH_NOISE_PATH} \\
            --lora_path_low {SVI_LOW_NOISE_PATH} \\
            --init_latents_path {temp_latents_path} \\
            --prompt "{prompt}" \\
            --cfg {scheduler.cfg} \\
            --steps {scheduler.get_steps()} \\
            --output_path {segment_output_path}
        """
        import subprocess, tempfile, json, os
        from pathlib import Path
        
        # Save latents to temp file for subprocess
        with tempfile.NamedTemporaryFile(suffix=".pt", delete=False) as f:
            import torch
            torch.save(init_latents, f.name)
            latents_path = f.name
        
        segment_output_path = f"/workspace/output/segments/segment_{segment_index:03d}.mp4"
        
        cmd = [
            settings.SVI_WAN22_PYTHON,
            str(settings.SVI_REPO_PATH / "inference_svi_2.0.py"),
            "--lora_path_high", str(settings.SVI_HIGH_NOISE_PATH),
            "--lora_path_low", str(settings.SVI_LOW_NOISE_PATH),
            "--init_latents_path", latents_path,
            "--prompt", prompt,
            "--cfg", str(scheduler.cfg),
            "--steps", str(scheduler.get_steps()),
            "--output_path", segment_output_path,
            "--camera_motion", composition_plan.camera_motion,
            "--motion_vector", composition_plan.motion_vector,
        ]
        
        result = subprocess.run(cmd, capture_output=True, text=True, timeout=600)
        os.unlink(latents_path)
        
        if result.returncode != 0:
            raise RuntimeError(f"SVI subprocess failed: {result.stderr}")
        
        return self._load_segment(segment_output_path, segment_index)
    
    def _validate_segment_triple(
        self,
        segment: "VideoSegment",
        buffer: TemporalBuffer,
        context: ImmutableContext,
        segment_index: int,
    ) -> None:
        """
        VALIDATION PROPAGATION CONTRACT: ALL THREE validators must pass.
        Raises TemporalSegmentFailureError if any fails.
        """
        # 1. CLIP identity validation (RULE-89)
        keyframe = self._extract_keyframe(segment)
        clip_score = self.clip_validator.score(
```

```python
"""
MotionStateTracker — Estimates velocity, direction, magnitude from TemporalBuffer.
FR-940–FR-944: optical-flow-based motion estimation.
MotionState propagated via context.evolve() to prevent motion reset between segments.
"""
from dataclasses import dataclass
from typing import Optional
import torch
from torch import Tensor
from vga.temporal.temporal_buffer_manager import TemporalBuffer
from vga.config.settings import settings

@dataclass(frozen=True)
class MotionState:
    """
    FR-941: typed dataclass with velocity_vector, direction, magnitude.
    Propagated via context.evolve() to prevent motion reset.
    """
    velocity_vector: Optional[tuple] = None   # (dx, dy) optical flow mean
    direction: str = "stationary"             # "forward", "backward", "left", "right", etc.
    magnitude: float = 0.0                    # scalar magnitude of motion
    segment_index: int = 0

class MotionStateTracker:
    """
    Uses optical flow (torchvision or opencv) to estimate motion from TemporalBuffer.
    
    FR-944: MotionState velocity_vector MUST be applied to each segment's prompt/plan
            to prevent motion reset between segments.
    SLA: estimate() MUST complete in ≤ 1.0 second (NFR-168).
    """
    
    def estimate(self, buffer: TemporalBuffer) -> MotionState:
        """
        Estimate MotionState from 5-frame buffer using optical flow.
        Computes flow between consecutive frame pairs, then aggregates.
        
        Returns: MotionState with velocity_vector, direction, magnitude
        """
        frames = buffer.frames  # (5, C, H, W) CPU
        
        # Compute optical flow between consecutive frames
        flows = self._compute_optical_flow(frames)  # (4, H, W, 2) flow fields
        
        # Aggregate flows to get mean velocity
        mean_flow = flows.mean(dim=(0, 1, 2))  # (2,) = (dx, dy)
        magnitude = float(mean_flow.norm())
        direction = self._classify_direction(mean_flow)
        
        return MotionState(
            velocity_vector=(float(mean_flow[0]), float(mean_flow[1])),
            direction=direction,
            magnitude=magnitude,
            segment_index=buffer.segment_index,
        )
    
    def _compute_optical_flow(self, frames: Tensor) -> Tensor:
        """Compute optical flow between consecutive frame pairs using torchvision."""
        # Use torchvision.models.optical_flow or opencv cv2.calcOpticalFlowPyrLK
        # frames: (5, C, H, W)
        # Convert to grayscale for optical flow
        # Return flows: (4, H, W, 2) for 4 pairs: (0→1, 1→2, 2→3, 3→4)
        ...
    
    def _classify_direction(self, flow: Tensor) -> str:
        """Classify dominant motion direction from mean flow vector."""
        dx, dy = float(flow[0]), float(flow[1])
        mag = (dx**2 + dy**2) ** 0.5
        
        if mag < settings.MOTION_STATIONARY_THRESHOLD:
            return "stationary"
        
        if abs(dx) > abs(dy):
            return "right" if dx > 0 else "left"
        else:
            return "forward" if dy > 0 else "backward"
```

```python
"""
VideoSegmentGenerator — Stage S-08: Generate Segment_1 using Wan2.2.
Owns: Segment_1 generation + TemporalBuffer initialization.
Does NOT own: Segments 2..N (owned by temporal_engine.py).
Spec: VGA SRD v17.2 §1.4, §3.67, §10.2
"""
from vga.agents.base_agent import BaseAgent
from vga.temporal.temporal_buffer_manager import TemporalBufferManager
from vga.validation.clip_validator import CLIPValidator
from vga.models.wrappers.wan_wrapper import WanWrapper
from vga.state.immutable_context import ImmutableContext
from vga.config.settings import settings
from vga.core.exceptions import TemporalBufferError
from vga.models.schemas import TemporalState
from vga.core.logger import get_logger

log = get_logger(__name__)

class VideoSegmentGenerator(BaseAgent):
    """
    Stage S-08: Generate Segment_1 from refined image using Wan2.2-I2V-A14B-FP8.
    
    Critical steps:
    1. Load refined image from S-07 output
    2. Assert CompositionPlan exists (RULE-88)
    3. Call Wan2.2 with CompositionPlan motion params
    4. Run CLIPValidator on Segment_1 keyframe (RULE-89)
    5. Initialize TemporalBuffer from Segment_1
    6. Assert buffer.frames.shape[0] == 5 (RULE-86)
    7. Write segment_001.mp4
    8. Return (segment_1, updated_context_with_buffer)
    """
    stage_id = "S-08"
    
    def run(self, input_data, context: ImmutableContext):
        context.assert_composition_plan()  # RULE-88
        context.assert_identity_frozen()   # char_identity_ref must be frozen
        
        # Generate Segment_1 using Wan2.2
        segment_1 = self.wan_wrapper.generate(
            init_image=input_data.refined_image,
            prompt=input_data.prompt,
            motion_vector=context.composition_plan.motion_vector,
            camera_motion=context.composition_plan.camera_motion,
        )
        
        # CLIP validation on keyframe (RULE-89)
        keyframe = segment_1.frames_tensor[len(segment_1.frames_tensor)//2]
        clip_score = self.clip_validator.score(
            keyframe, context.identity_state.embedding_vector
        )
        if clip_score < settings.CLIP_IDENTITY_THRESHOLD:
            raise ValueError(f"Segment_1 CLIP score {clip_score:.3f} < {settings.CLIP_IDENTITY_THRESHOLD}")
        
        # Initialize TemporalBuffer (RULE-86)
        buffer = TemporalBufferManager.init(segment_1, context.scene_id)
        # Buffer assertion happens inside init() — raises TemporalBufferError if violated
        
        # Write segment file
        segment_path = f"/workspace/output/{context.job_id}/{context.scene_id}/segment_001.mp4"
        segment_1.save(segment_path)
        
        log.info("segment_1_generated", scene_id=context.scene_id, clip_score=clip_score)
        log.info("temporal_buffer_initialized", frame_count=5)
        
        # Evolve context with temporal state
        new_context = context.evolve(
            temporal_state=TemporalState(buffer=buffer, segment_index=1),
        )
        
        return segment_1, new_context
```

- [ ] `TemporalBufferManager.init()` raises `TemporalBufferError` if segment has < 5 frames
- [ ] `TemporalBufferManager._assert_buffer_size()` raises immediately if frames.shape[0] ≠ 5
- [ ] `TemporalBufferManager.encode()` transfers to GPU and returns CPU tensor
- [ ] `SVIScheduler(total_steps=50, cfg=6.5)` raises `SVICFGViolationError`
- [ ] `SVIScheduler(cfg=5.5).get_lora_weight(step=0)` returns 0.6 (high noise phase)
- [ ] `SVIScheduler(cfg=5.5).get_lora_weight(step=40)` returns 0.4 (low noise phase for 50-step scheduler)
- [ ] `TemporalEngine.generate_segments()` uses an explicit for loop, never processes segments in batch
- [ ] SVI invocation always goes through subprocess with `SVI_WAN22_PYTHON`
- [ ] `MotionStateTracker.estimate()` returns `MotionState` with velocity_vector, direction, magnitude
- [ ] Single-frame latents (shape (1, C', H', W')) passed to SVI raises `AutoregressiveViolationError`
- [ ] `VideoSegmentGenerator.run()` with missing CompositionPlan raises `CompositionPlanValidationError`
